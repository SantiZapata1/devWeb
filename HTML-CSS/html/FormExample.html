Formularios

Para realizar formularios se usan diversos elementos:

  1. FORM es el elemento tipo block donde va todo el contenido, para enviar datos, este puede usar 2 metodos que debe indicar:
    1. GET: El método GET se utiliza para solicitar datos de un recurso específico en un servidor. Cuando se envía una solicitud GET, los datos se envían como parte de la URL en la cadena de consulta (query string). Por ejemplo, si tienes un formulario con un campo de texto llamado nombre y envías los datos usando GET, la URL resultante podría ser algo como http://ejemplo.com/procesar.php?nombre=Juan. Este método es útil para solicitar recursos o datos específicos y se pueden marcar como favoritos o compartirse fácilmente, ya que los parámetros se muestran en la URL.
    2. POST: El método POST se utiliza para enviar datos al servidor para ser procesados. A diferencia de GET, los datos enviados con POST no se muestran en la URL y se envían en el cuerpo de la solicitud HTTP. Esto hace que sea más seguro para enviar datos sensibles como contraseñas. Además, no hay límite en la cantidad de datos que se pueden enviar con POST, mientras que GET tiene limitaciones en la longitud de la URL. POST es comúnmente utilizado para enviar datos de formularios y para realizar actualizaciones en el servidor.
    En resumen, GET se utiliza para solicitar datos, mientras que POST se utiliza para enviar datos al servidor para ser procesados. La elección entre GET y POST depende del tipo de datos que se están enviando y del propósito de la solicitud.
  2. INPUT: son los elementos en donde nosotros ingresaremos diferente tipo de informacion. Este tiene varios atributos:
    • type: indica el tipo de dato que ingresaremos pueden ser de texto, password, checkbox, radio, submit, file, date, email, etc.
    • placeholder: es el texto que esta dentro del input indicando que debe hacer el usuario en ese espacio
    • name:El atributo name en un elemento input de HTML se utiliza para darle un nombre al campo de entrada. Este nombre es importante porque se utiliza para identificar el campo cuando se envía el formulario al servidor, ya sea mediante el método GET o POST.
    • value: se utiliza para especificar el valor inicial del campo de entrada. Este valor se mostrará en el campo cuando la página se cargue por primera vez y puede ser modificado por el usuario.
    • required: Indica que el control de entrada es obligatorio y no se puede enviar el formulario si está vacío. Este atributo es especialmente útil para campos de texto y otros controles que requieren entrada del usuario.
    • disabled: Indica que el control de entrada está deshabilitado y no se puede utilizar. Los controles deshabilitados no se envían cuando se envía el formulario.
