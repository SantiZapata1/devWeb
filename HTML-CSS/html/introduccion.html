Introduccion:

CSS es un lenguaje de definicion de los estilos de los elementos html de una pagina web. Define el estilo o como se veran los distintos elementos como titulos, parrafos, secciones, imagenes, etc. La combinacion de HTML y CSS permiten crear paginas web estaticas. (solo lectura)

Selectores basicos

Un selector es la representacion de un objeto al que nosotros le podemos aplicar los estilos, existen muchos, pero los 3 principales son:

  1. Selector de clase: Las clases generales son como plantillas con ciertos atributos especificos que se pueden asignar a distintos objetos, la ventaja de usarlas es que cada elemento puede implementar muchas clases.
    Es importante el buen uso de las clases ya que su buena implementacion nos permitira reutilizar codigo y adaptarnos facilmente a diferentes frameworks.
    Debemos poder nombres descriptivos a las clases, que las clases sean generales y no tan especificas, para asi mediante las combinaciones de diferentes clases se pueden disenar distintos elementos.
  2. Selector de id: Usando IDs para dar estilos es el mismo procedimiento que las clases, con la diferencia de que un solo elemento puede tener ese ID, no mas. dar estilos de esta forma # se considera una mala practica dar estilos de esta forma
  3. Selector de etiquetas: Usando los nombres de las etiquetas para dar estilos le aplicamos el mismo estilo a TODOS esos elementos del documento.

Bloque de declaraciones {}: Al margen de que selector usemos, cada uno tiene un espacio en donde declaramos el valor de cada atributo de cada selector

ejemplo: 

/* etiquetas code y pre */
            code,pre{
                color: rgb(200, 200, 200);
                background-color: rgb(30, 30, 30);
            }

selectores avanzados

  1. hijos directos

    Estos selectores se usan para darle estilo a aquellos elementos que estan dentro de otro, pero son los primeros, es decir que no hay ningun elemento entre estos y su elemento padre. Hay que tener en cuenta que si usamos este selector y el hijo no es directo no funcionara. Este selector se representa con un ">"

    ejemplo:
    ol > li { }
    1. primavera
    2. verano
    3. invierno

    Como los list items son los hijos directos del contenedor ol, se les aplican los estilos asignados.

  2. hijos descendientes

    En este caso, son los elementos que tienen elementos entre ellos y su elemento padre, es decir que no son hijos directos. Se representan con un espacio en blanco.

    ejemplo
    ol b{ }
    1. primavera aaaaa
    2. veranoaaaaaaa
    3. inviernoaaaaaa

    Los elementos b que esten en algun lugar dentro del ol, absorveran los atributos asignados.

  3. selector de hermanos (general)

    El combinador ~ separa dos selectores hermanos y selecciona el segundo elemento sólo si está precedido por el primero y ambos comparten un padre común. No hace falta que esten de forma consecutiva.

    ejemplo
    p ~ span {
                    color: red;
                    font-size: 14px;
                }

    Se hace referencia a este selector como selector adyacente o selector del próximo hermano. Sólo seleccionará un elemento especificado que esté inmediatamente después de otro elemento especificado.

  5. selectores de atributos

    se usan para dar estilos a todos los elementos que tengan un atributo especifico que le indiquemos nosotros. En general, hay diferentes formas de aplicar estilos a elementos que cumplan con ciertas caracteristicas.
    Su implementacion se realiza colocando el selctor del elemento principal seguido de corchetes que en su interior tendran la condicion que deben cumplir para que se apliquen los estilos.

    algunos ejemplos de selectores por atributos son:

    • Elemento "a" que tenga el atributo title
    • Elemento "a" que coincida con cierta URL
    • Elemento "a" que su URL contenga cierta letra o palabra
    • Elemento "a" que su URL comience con cierta letra o palabra
    • Elemento "a" con un href que termine en ".org"
    • Elementos "a" cuyo atributo class contenga la palabra "logo"

  6. selector universal

    Va a aplicar el estilo a todos los elementos html del documento, se designa con el simbolo "*". Tambien se la puede usar combinado con otra etiqueta para aplicar a todos los elementos que esten contenido en otro elemento

    ejemplo
    ol * {}
    1. primavera
    2. verano
    3. invierno

Pseudo-clases

Las pseudoclases en CSS son selectores que te permiten seleccionar y estilizar un elemento en función de su ESTADO o POSICION dentro del documento HTML.
Estos estados pueden ser interacciones del usuario, como "hover" (cuando el mouse está sobre el elemento), "active" (cuando el usuario hace clic en el elemento) o "visited" (cuando el enlace ya ha sido visitado). También hay pseudoclases que se aplican a ciertos tipos de elementos, como "first-child" (primer hijo) o "nth-child" (ésimo hijo).

sitaxis:
selector:pseudoclase { propiedad: valor; }

Si el objeto esta en el estado o posicion que se indica, adoptara los atributos que le indiquemos

Pseudo-elementos

Al igual que las Pseudo-classes, los pseudo-elementos se añaden a los selectores, pero en cambio, no describen un estado especial sino que, permiten añadir estilos a una parte concreta del documento.
Por ejemplo, el pseudoelemento ::first-line selecciona solo la primera línea del elemento especificado por el selector.
En este caso, no importa el estado sino su UBICACION en el objeto

agrupar selectores

Cuando se desea asignar el mismo estilo a diferentes elementos se usa la coma "," para indicar que el bloque de propiedades se le asigna a todos los elementos que pongamos

ejemplo:

.form-agrupar-selectores input[type="email"],
.form-agrupar-selectores input[type="submit"]

display: block;
width: 200px;

}

El algoritmo de css

Es la forma en la que el navegador aplica los estilos CSS al documento HTML, los podemos dividir en:

  1. La cascada:

    De forma secuencial en el siguiente orden, se le van aplicando estilos a los elementos del documento html:

    1. User agent: Los primeros estilos que le aplican son los que estan predeterminador por el navegador que estemos usando
    2. personalizaciones del usuario: Luego las configuracion del navegador del usuario
    3. Estilos del autor: Por ultimo el codigo css del programador

  2. La espeficidad

    Continuando con el ultimo punto anterior "estilos del autor", estos a su vez dependen de la espeficidad, que vienen de peso especifico, cada elemento html tiene su propio peso, y los estilos css tambien dependen del mismo, ganan siempre los de mayor peso especifico. Segun que selector usemos, se le dara mayor o menor importancia al estilo indicado.
    Es muy importante el peso del selector ya que cuando hay conflictos de estilos css quedara finalmente el mas grande.

    Colocando el mouse sobre el selector css, veremos su peso especifico como muestra la imagen de arriba

    Usar !important se considera una mala practica porque rompe con la especificidad, ya que un estilo dado con ese selector se aplicara de forma obligatoria.

    Ejemplo:
    aside{
    background-color: yellowgreen !important;
}
.ejemplo-espe {
    background-color: lightblue;
}

#ejemplo-espe{
    background-color: #197e13 ;
}
    En este ejemplo, al texto se le aplican diferentes background-color con el fin de que quede el de mayor peso especifico, se usan ID, clase, estilos en linea y el atributo !important.

  3. La herencia

    Es la capacidad de que un selector herede los estilos de su elemento padre, y esto se puede lograr asignandole la propiedad inherit a cada atributo que deseamos heredar. Si deseamos evitar la herencia, esta quedaria cancelada si le asignamos otros valores a nuestro selector o si le colocamos la propiedad initial a cada atributo.

    Ejemplo de herencia:
    /* Cada item de la lista hereda el borde que tiene su contenedor padre "ol" */

.ejemplo-herencia{
    border: thin solid black;
}
.ejemplo-herencia li{
    border: inherit;
}

Normalize

Normalize es un fichero CSS que se lo linkea asi link rel="stylesheet" href="css/normalize.css", cuyo principal objetivo es mantener los estilos similares en los distintos navegadores, ya que cada uno agrega sus propios estilos por defecto. Por lo que esta hoja de estilo describe los estilos "estandares" que tendra cada elemento HTML.

Ejemplo
h1 {
font-size: 2em;
margin: 0.67em 0;
}

Prefijos

Los proveedores de navegadores a veces agregan prefijos a las propiedades de CSS experimentales o no estándar, por lo que los desarrolladores pueden experimentar con nuevas ideas mientras que, evitan que se confíe en sus experimentos y luego rompan el código de los desarrolladores web durante el proceso de estandarización. Los desarrolladores deben esperar para incluir la propiedad sin prefijar hasta que se estandarice el comportamiento del navegador.

Los principales navegadores usan los siguientes prefijos:

  1. -webkit- (Chrome, Safari, nuevas versiones de Opera, casi todos los navegadores iOS (incluyendo Firefox for iOS); basicamente, cualquier navegador basado en WebKit)
  2. -moz- (Firefox)
  3. -o- (Versiónes antiguas, pre-Webkit, de Opera)
  4. -ms- (Internet Explorer y Microsoft Edge)

SEO

SEO (Search Engine Optimization) es el proceso de mejorar la visibilidad de un sitio web en los resultados orgánicos (no pagados) de los motores de búsqueda. El objetivo del SEO es aumentar la cantidad y calidad del tráfico que llega a un sitio web, al mejorar su posición en los resultados de búsqueda para ciertas palabras clave relevantes. El SEO implica una serie de prácticas, tanto en el contenido del sitio web como en aspectos técnicos y de estructura, con el fin de que los motores de búsqueda comprendan mejor de qué trata el sitio y lo consideren relevante para ciertas consultas de búsqueda. Algunas técnicas comunes de SEO incluyen la optimización de palabras clave, la creación de contenido de alta calidad, la optimización de la velocidad de carga del sitio, la mejora de la experiencia del usuario y la construcción de enlaces de calidad.

Factores que influyen el seo:

  • Semantica HTML: Si bien hay elementos que se visualizan igual en el documento html, cada uno tiene una funcion semantica segun el contenido que lo compone, esto es importante utilizarlo bien para lograr que el posicionamiento en los motores de busqueda mejore.
