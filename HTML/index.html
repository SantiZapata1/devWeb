etiquetas semanticas estructurales

Todas estas etiquetas contenedoras sirven para almacenar informacion, pero cada una tiene su funcion semantica, osea que segun el contenido que neesitamos almacenar usaremos una u otra.
no hay una sola forma de usarlo, ni tienen que haber una sola de cada una por pagina, estas se combinan segun nuestras necesidades

Tabla periodica de las diferentes etiquetas HTML

  • div Cuando necesitas agrupar y contener elementos sin un significado semántico específico. El div es una etiqueta de propósito general y se utiliza comúnmente para aplicar estilos o manipular contenido con JavaScript
  • header Para definir la cabecera de una sección o de la página en general. Puede contener elementos como títulos, logotipos y enlaces de navegación
  • main Para contener el contenido principal de la página (excluyendo la cabecera, pie, etc.). Deberías tener una sola instancia de main en tu documento.
  • footer Para representar el pie de una sección o de la página en general. Puede contener información de derechos de autor, enlaces de contacto, etc.
  • nav Para definir un conjunto de enlaces de navegación, como menús y barras de navegación.
  • article Para definir un contenido independiente y autocontenido, como un artículo o publicación. Deberías usar article cuando el contenido pueda existir de manera independiente y tenga su propia identidad.
  • aside Para contenido relacionado pero que es tangencial al contenido principal, como barras laterales, anuncios, etc.
  • section Para agrupar contenido relacionado dentro de una sección. Deberías usar section cuando tengas contenido que forme una unidad temática

La clave es utilizar estas etiquetas de manera semántica para estructurar tu documento de una manera significativa y comprensible tanto para los desarrolladores como para las tecnologías de asistencia y los motores de búsqueda. Al elegir la etiqueta correcta para cada parte de tu página, estás comunicando claramente la relación y el propósito de cada sección de tu contenido.

formularios

El HTML de un formulario web está compuesto por uno o más controles de formulario, pueden ser campos de texto de una o varias líneas, cajas desplegables, botones, casillas de verificación o botones de opción, y se crean principalmente con el elemento input, aunque hay algunos otros elementos que también hay que conocer.
Al elemento input se le agregan diferentes atributos para adaptarlo a nuestras necesidades

ejemplo de input sin atributos:

atributos

  • type tipo de dato del input, mas abajo se desarrollan los diferentes inputs
  • placeholder texto predeterminador que esta cuando el input esta vacio
  • value El valor proporcionado por el atributo value es el valor que se enviará al servidor cuando se envíe el formulario. Es el valor predetenerminado del input, ejemplo "si"
  • name se utiliza para identificar y referenciar los datos enviados cuando se envía un formulario. Este atributo asigna un nombre a la entrada de datos para que pueda ser identificado por el servidor cuando el formulario se envía.
  • action especifica la URL a la que se enviarán los datos del formulario cuando el usuario lo envíe
  • method Especifica el método HTTP utilizado para enviar los datos del formulario. Los dos métodos más comunes son "GET" y "POST". La elección del método afecta cómo se transmiten y procesan los datos. Cuando se utiliza "GET", los datos se adjuntan a la URL, mientras que "POST" envía los datos en el cuerpo de la solicitud
  • id identificador de la etiqueta
  • entre otros, pero esos son los mas importantes xd.

Patterns

Las expresiones regulares (también conocidas como regex o regexp) son patrones de búsqueda utilizados para realizar operaciones de búsqueda y manipulación de cadenas de texto. Aunque no son específicas de HTML, se pueden usar en combinación con HTML para realizar búsquedas y manipulaciones más avanzadas en el código HTML.

etiquetas meta

Las metaetiquetas, etiquetas meta o elementos meta (también conocidas por su nombre en inglés, meta tags) son etiquetas HTML que se incorporan en el encabezado de una página web y que resultan invisibles para un visitante normal, pero de gran utilidad para navegadores u otros programas que puedan valerse de esta información. Su propósito es el de incluir información (metadatos) de referencia sobre la página: autor, título, fecha, palabras clave, descripción, etc. Características Las meta tags son unas etiquetas pertenecientes al HTML que se deben escribir dentro del tag general head y que lo podemos definir como líneas de código que indican a los buscadores que indexan la página por qué términos debe ser encontrada. Dependiendo de la utilización, caracterización y objetividad de dichos meta, más otros factores SEO se puede conseguir una excelente posición en el listado resultante de una búsqueda. Según varios estudios,1​ cuando un navegante realiza una búsqueda no suele ir más allá del resultado 20, pues entre estos primeros puestos suele encontrar aquello que busca. Para quedar entre estos puestos es conveniente tener unos metas optimizados. 2​

accesibilidad web

La accesibilidad web se refiere a la práctica de diseñar y desarrollar sitios web y aplicaciones de manera que sean accesibles y utilizables por todas las personas, independientemente de sus capacidades o discapacidades. El objetivo es garantizar que todas las personas, incluyendo aquellas con discapacidades visuales, auditivas, motoras o cognitivas, puedan interactuar de manera efectiva con el contenido en línea. Para garantizar la accesibilidad web, se han desarrollado pautas y estándares, como las Pautas de Accesibilidad al Contenido Web (WCAG, por sus siglas en inglés), que proporcionan criterios y recomendaciones específicas para crear contenido web accesible. Estas pautas son ampliamente adoptadas y seguidas por desarrolladores y diseñadores web en todo el mundo.

Preocuparnos por estas cosas le da mucha calidad al sitio web, ya que las hace inclusivas para todas las personas, es una buena practica darle importancia a estos detalles

Estas practicas las implementamos con multiples atributos en las etiquetas como por ejemplo: no le di bola a esto jjjje

ejemplos:

  1. roles
  2. index
  3. area label
  4. etc

Cada una sirve para hacer que cada parte sea inclusiva para las distintas discapacidades
